Public-Key Cryptography

Public-Key Encryption

Public-key cryptography uses two keys: a public key for encryption and a private key for decryption. It enables secure communication over insecure channels.

Definitions

Public-key encryption consists of three algorithms:

  • Key Generation: Generates a public and private key pair.
  • Encryption: Uses the public key to encrypt a message.
  • Decryption: Uses the private key to decrypt a ciphertext.

RSA Cryptosystem

The RSA algorithm relies on the difficulty of factoring large composite numbers. It uses the following steps:

  1. Generate two large primes p and q, and compute n = p * q.
  2. Choose a public exponent e and compute the private key d such that (e * d) % φ(n) = 1.
  3. Encrypt: C = M^e % n
  4. Decrypt: M = C^d % n
Example:

Suppose p = 61, q = 53, and e = 17. Compute n = 61 * 53 = 3233 and find d such that (17 * d) % φ(3233) = 1.

ElGamal Cryptosystem

The ElGamal encryption scheme uses the hardness of the discrete logarithm problem. It operates in a finite cyclic group G.

Cramer-Shoup Encryption

The Cramer-Shoup scheme is a practical, efficient, and secure encryption algorithm resistant to adaptive chosen ciphertext attacks.

Digital Signatures

Digital signatures ensure message integrity and authenticity by enabling the recipient to verify the sender's identity.

Definitions

A digital signature scheme consists of:

  • Key Generation: Produces a signing key and a verification key.
  • Signing: Uses the signing key to generate a signature for a message.
  • Verification: Uses the verification key to confirm the signature is valid.

Full-Domain Hash

This technique hashes a message before signing it, ensuring security under the random oracle model.

Waters' Signatures

Waters' signatures are efficient signature schemes based on assumptions about bilinear maps and are useful in identity-based systems.

Identification Schemes

Identification schemes verify the identity of a party in a communication. They often serve as a foundation for constructing digital signatures.

Constructions

Common constructions involve zero-knowledge proofs, where one party proves knowledge of a secret without revealing it.

Applications

Identification schemes are used in secure login systems and digital signature generation.

Identity-Based Encryption (IBE)

Identity-Based Encryption allows the use of identities, such as email addresses, as public keys. The private keys are generated by a trusted authority.

Applications

IBE is useful in systems requiring certificate-less encryption and streamlined key management.
