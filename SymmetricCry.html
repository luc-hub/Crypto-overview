Pseudorandom Generators (PRG)

A Pseudorandom Generator (PRG) is an algorithm that expands a small random seed into a longer sequence of bits that appears random. PRGs are foundational in cryptography, providing efficiency while maintaining security.

Key Concepts

  • Hard-core bits: Bits of the output that are computationally infeasible to predict from the input seed.
  • PRG Constructions: Often based on mathematical problems such as modular exponentiation or lattice-based cryptography.

Pseudorandom Functions (PRF)

A Pseudorandom Function (PRF) is a deterministic function that simulates a random function when keyed. It is used in many cryptographic protocols to ensure secure, random-like outputs.

PRF Constructions

  • Feistel Networks: A common construction for PRFs, using multiple rounds of mixing and permutations.
  • Applications: MACs, stream ciphers, and secure encryption schemes.

Symmetric Encryption

Symmetric encryption uses the same key for both encryption and decryption. Popular algorithms include AES (Advanced Encryption Standard) and DES (Data Encryption Standard).

Modes of Operation

  • ECB: Encrypts blocks independently (not secure for patterns).
  • CBC: Adds chaining between blocks for enhanced security.
  • CTR: Converts a block cipher into a stream cipher.

Message Authentication

Ensuring data integrity and authenticity is crucial. Message Authentication Codes (MACs) and authenticated encryption combine encryption with integrity checks.

Definitions and Constructions

  • HMAC: Hash-based MAC, often using SHA-256.
  • Authenticated Encryption: Combines encryption with a MAC (e.g., GCM mode in AES).
Example: HMAC-SHA256(key, message)

Hash Functions

Hash functions compress data into fixed-size digests. They are fundamental in cryptography for data verification.

Key Properties

  • Random Oracle Model: Theoretical model treating hash functions as truly random mappings.
  • Collision Resistance: It is computationally infeasible to find two inputs with the same hash.

Merkle-Damgaard Construction

A common construction technique for iterative hash functions, where input blocks are processed sequentially.
