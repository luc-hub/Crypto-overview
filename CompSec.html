Understanding the Foundations of Modern Cryptography

1. One-Way Functions (OWF) and Complexity Theory

Definition: A one-way function (OWF) is a function f: X → Y that is easy to compute but hard to invert. This means that given x, computing f(x) is efficient, but given y, finding any x such that f(x) = y is computationally infeasible.

One-way functions are the cornerstone of computational security, ensuring that certain tasks (e.g., breaking encryption) remain practically impossible even with powerful computers.

2. Brush-up on Number Theory

Number theory provides the foundation for many cryptographic schemes. Key concepts include modular arithmetic, prime factorization, and discrete logarithms.

Candidate OWF

  • Factoring: Given N = p * q (where p and q are large primes), finding p and q is computationally hard.
  • RSA: Based on the difficulty of inverting modular exponentiation.
  • Discrete Logarithm (DL): Given y = g^x mod p, finding x is hard.
  • Lattice-based (LWE): Hardness based on the Learning With Errors problem, resilient even against quantum attacks.
Example: RSA Encryption 
Key generation:
1. Choose primes p, q.
2. Compute N = p * q and φ(N) = (p-1)(q-1).
3. Choose e such that gcd(e, φ(N)) = 1.
4. Compute d such that e*d ≡ 1 (mod φ(N)).

Public key: (N, e)
Private key: (N, d)

Encryption: c = m^e mod N
Decryption: m = c^d mod N

3. Computational Indistinguishability and Decisional Assumptions

Computational indistinguishability ensures that two distributions are indistinguishable by any efficient algorithm. Decisional assumptions strengthen this concept by proposing specific hard problems.

Key Assumptions

  • DDH (Decisional Diffie-Hellman): Given (g, g^a, g^b), it is hard to distinguish g^(ab) from a random value.
  • LWE (Learning With Errors): Distinguishing noisy linear equations from random ones is hard.
